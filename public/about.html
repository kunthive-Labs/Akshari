A specimen book, not a font dump.

Akshari exists so you can choose type by how it feels and where it's going to live, not by scrolling a flat A-to-Z list and guessing.

What Akshari does

Every family is shown as a live, readable specimen rather than a thumbnail: full alphabet, figures, and your own sample text, rendered in the real webfont. You can describe the job the type needs to do - "warm, trustworthy fintech" or "technical dev tool" - and Akshari ranks the catalog against that description. Filters, curated directions, and a side-by-side compare view sit alongside it, and every family ends in one click: copyable CSS for the exact weights shown.

The catalog

Akshari indexes the full Google Fonts library - 1,900+ families and growing - sourced directly from Google's font metadata and kept in sync as new families are published. Every included typeface keeps its own license, normally the SIL Open Font License (OFL). Akshari doesn't modify font files; it only adds discovery metadata on top.

How families get tagged

Search works on feeling because each family is scored against a small, fixed vocabulary of semantic tags (things like modern, editorial, technical, rounded) rather than free-text keywords. Tags come from a repeatable pipeline: deterministic OpenType metrics are extracted from each font file, a rendered specimen and those measurements are reviewed against the fixed vocabulary, and every accepted tag is checked against a hand-curated calibration set before it ships. The pipeline and vocabulary are versioned in the open-source repository, so the scoring logic is never a black box.

Open source and self-hosting

Akshari's application code is MIT licensed and built by Kunthive Labs. The whole stack - interface, catalog export, and tagging pipeline - is designed to run on your own infrastructure.

git clone the repository, then docker compose up --build to run a self-hosted instance, or read the API and pipeline docs in the README to import your own catalog.
